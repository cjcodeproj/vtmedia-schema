Elements

Primary Element: Catalog

Element<catalog>
Module vtm-mod-all-catalog.xsd
UsageOptional, one occurence max
Parent Element <movie>
Child Elements <identification> <altTitles> <copyright> <extRefs>

The catalog element is for all information related to the title, the registration, and outside references to the film. It allows for a local movie ID for the purpose of maintaining a localized library system, and reference data to external libraries. It also tracks copyright registrations, and alternate titles or title changes tied to the movie.

Identifcation Element

Element<identification>
Module vtm-mod-all-catalog.xsd
UsageOptional, one occurence max
Parent Element <catalog>
Child Elements <id> <uuid>

The identification element holds all information tied to a set of unique, local reference key to the movie. Most instances will only have one child element, but it is possible to have multiple identification keys. At the very least, the element must contain one id element or one uuid element.

Element<id>
Module vtm-mod-all-catalog.xsd
UsageOptional, multiple occurences
Valuesimple string (xs:token)
Parent Element <identification>
Child ElementsNone
AttributesNameTypeUsage
typestring (xs:token)optional
applicationstring (xs:token)optional

The id element holds a unique identifier value for the movie. The type and application attributes are designed to allow for multiple id values for the same movie, which could be utilized by external tools or applications.

Element<uuid>
Module vtm-mod-all-catalog.xsd
UsageOptional, one occurence max
ValueUUID (custom)
Parent Element <identification>
Child Elements None

The uuid element stores a UUID value, which could be used as a unique identifier for a movie. Only one uuid element is allowed, as it's assumed that only one is necessary if it is used at all.

The UUID conformity check is based on a regex pattern, and does not adhere 100% to the UUID format as far as checking for UUID version 1 or UUID version 4. The generation of the UUID value is up to the implementor.

Alternate Titles

Element<altTitles>
Module vtm-mod-all-catalog.xsd
UsageOptional, one occurence max
Parent Element <catalog>
Child Elements <originalTitle> <variantTitle> <alternateTitle> <productionTitle> <distributionTitle>

The altTitles element contains data on alternate titles the movie may be recognised as; or variant expressions of the current title that may be more suitable for readability, indexing, or speech to text operations.

Element<originalTitle>
Module vtm-mod-all-catalog.xsd
UsageOptional, one occurence max
Valuetitle string (xs:token)
Parent Element <altTitles>
Child Elements None

The originalTitle element covers rare situations when the name of a movie actually changes after release.

Movie TitleMovie YearOriginal Title
Live. Die. Repeat.2014Edge Of Tomorrow
Element<variantTitle>
Module vtm-mod-all-catalog.xsd
UsageOptional, one occurence max
Valuetitle string (xs:token)
Parent Element <altTitles>
Child Elements None

The variantTitle is designed to match the current main title of the movie, but with an alternate representation. It is only used in cases where the actual title is unusable for operations like index sorting, or text to speech conversion. Titles that include numbers in the name, non-alphaetic characters, or emojii should have a variantTitle defined to convey what the title actually says.

Movie TitleMovie YearVariant Title
Se7en1995Seven
Element<productionTitle>
Module vtm-mod-all-catalog.xsd
UsageOptional, one occurence max
Valuetitle string (xs:token)
Parent Element <altTitles>
Child Elements None

The productionTitle is used for cases where a movie is produced under a different title. The reasoning behind a production title could be related to security during shooting, or post production.

Copyright Information

Element<copyright>
Module vtm-mod-all-catalog.xsd
UsageOptional, one occurence max
Parent Element <catalog>
Child Elements <year> <holders> <country> <filing> <registrationNo> <expires>

The copyright element details the copyright information tied to the movie. It records the year of the copyright, the holders, the registration number and filing information, and the year the copyright expires.

Element<year>
Module vtm-mod-all-catalog.xsd
UsageRequired, one occurence max
Valueyear number (xs:gYear)
Parent Element <copyright>
Child Elements None

The year element records the year of the copyright.

Element<holders>
Module vtm-mod-all-catalog.xsd
UsageOptional, one occurence max
Parent Element <copyright>
Child Elements <holder>

The holders element tracks all holders of a given copyright, with one or more holder child elements, each containing a single entity recorded as the holder of the copyright.

Element<holder>
Module vtm-mod-all-catalog.xsd
UsageRequired, at least one occurence
Valueentity name (xs:token)
Parent Element <holders>
Child Elements None

Each holder element contains a single holder entity for a given copyright.

Element<filing>
Module vtm-mod-all-catalog.xsd
UsageOptional, one occurence max
Valuefiling entity (xs:token)
Parent Element <copyright>
Child Elements None

The filing element identifies an organization where the copyright was registered.

Element<registrationNo>
Module vtm-mod-all-catalog.xsd
UsageOptional, one occurence max
Valuestring value (xs:token)
Parent Element <copyright>
Child Elements None

The registration number refers to the reference of the copyright as reported by the organization that handled the copyright registration.

Element<expires>
Module vtm-mod-all-catalog.xsd
UsageOptional, one occurence max
Valueyear (xs:gYear)
Parent Element <copyright>
Child Elements None

The year when the copyright is expected to expire.

External References

Element<extRefs>
Module vtm-mod-all-catalog.xsd
UsageOptional, one occurence max
Parent Element <catalog>
Child Elements <libraryRecord>

The extRefs element is used to list external references to the movie, usually library references.

Element<libraryRecord>
Module vtm-mod-all-catalog.xsd
UsageOptional, one or more occurences
Parent Element <extRefs>
Child Elements <library> <catalogNo> <link>

Contains a single library reference to a movie.

Element<library>
Module vtm-mod-all-catalog.xsd
UsageRequired, one occurence max
Valuelibrary name (xs:token)
Parent Element <libraryRecord>
Child Elements None

The library element contains the name of the library the record is from.

Element<catalogNo>
Module vtm-mod-all-catalog.xsd
UsageRequired, one or more occurences
Valuesimple string (xs:token)
Parent Element <libraryRecord>
Child ElementsNone
AttributesNameTypeUsage
typestring (xs:token)required

The catalogNo element holds the unique reference to the movie used by the external library. Multiple instances of the element can be used in case the library has multiple filing records or methods for the same film. The type identifies what type of reference is used by the library.

Element<link>
Module vtm-mod-all-catalog.xsd
UsageOptional, one occurence max
Valueuri (xs:anyURI)
Parent Element <libraryRecord>
Child Elements None

Contains an external link to the library record.
