Media Schema Project

The media schema is a data format to facilitate the recording of media (such as movies, television shows, or music) in a local data store.

Project Goals

The media schema project is designed to achieve the following goals.

  • Allow for the creation of a local dataset for physical media, whether it's a DVD, CD, book, or tape.
  • Record both the contents of the media, as well as data pertaining to ownership, and filing.
  • Allow for simple data entry, debugging, and validation using currently existing tools.
  • Allow for the easy exchange, publication or transfer of data between users using existing protocols.
  • Use a data schema that is flexible, easy to understand, and extensible.
  • Data is stored in a regular filesystem; no proprietary database is rquired.
  • The schema should use standard structures suitable for the attributes common to the media being recorded.
  • Data should be recorded in standard structures that allow for searching, indexing, and comparison operations.

The Data Structure

The media schema is an XML schema with two important tags: <medialist> and <contents>.

The <medialist> is the root element for storing data on one or more physical pieces of media, such as vinyl albums, books, a DVD boxed set, or film reels.

The <contents> element is for the describing the artistic work, whether it's a song, story, or a movie. The child elements underneath <contents> are maintained in separate schemas for improve maintainability and due to the different characteristics of the media. Each schema for the contents falls under a separate XML namespace.

ElementNamespace
<medialist>http://vectortron.com/xml/media/media
<movie>http://vectortron.com/xml/media/movie

Repository Files

Example Data

This repositoty contains a examples/ directory to illustrate how the schemas are used.

The example data directory is designed to be kept small enough to be managable, but still provide a strong cross section of data showing how the schema can be used.

Anyone using these tools to build their own dataset should create a separate directory/repo structure outside of this repository.

Test Code

There is a test directory with shell code for simple XML schema validation tests which can be used against the example data or any other data created by the user.
